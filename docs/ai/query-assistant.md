# AI-Enhanced Query Assistant

The AI-Enhanced Query Assistant makes the Plugs framework AI-native by allowing developers to query database models using natural language.

## Features

### 1. Natural Language to SQL

Query your models using English prompts. The engine converts these prompts into optimized, safe SQL statements.

### 2. Schema-Aware Context

The AI is automatically provided with your model's schema defined via `HasSchema`. This ensures the generated SQL uses precise column names, types, and relationships.

### 3. Fluent Integration

The `ai()` method returns a standard `QueryBuilder` instance, allowing you to chain further operations like `with()`, `where()`, `limit()`, or `get()`.

### 4. Developer Review Mode

Inspect AI-generated SQL before it executes by enabling the review mode.

## Usage

### Simple Natural Language Query

```php
$users = User::ai("Find all users registered in the last 7 days")->get();
```

### Complex Analytical Queries

```php
$topCustomers = User::ai("Find top 10 customers based on total order value")->get();
```

### Chaining Methods

```php
$activeUsers = User::ai("Show me active users")
    ->with('profile')
    ->limit(5)
    ->get();
```

## Security & Safety

The AI Query Assistant is built with security as a first-class citizen to protect your application and data.

### 1. SQL Injection Protection

The assistant **enforces parametric binding**. The AI is strictly instructed to return values as named parameters (e.g., `:param_0`) rather than embedding them in the query string. These parameters are then safely bound using PDO, neutralizing SQL injection attacks.

### 2. Read-Only Enforcement

To prevent accidental or malicious data modification, the engine **strictly validates** the generated SQL.

- Only `SELECT` statements are permitted.
- Any attempt to generate `UPDATE`, `DELETE`, `INSERT`, `DROP`, or other modification queries will result in a security exception before any database contact is made.

### 3. Schema Sandboxing

The AI only receives the schema for the model being queried. This helps ensure that the generated queries are restricted to the intended data context.

### 4. Developer Review

By enabling `AI_QUERY_REVIEW`, you can inspect every query generated by the AI in your logs. This transparency allows you to audit the assistant's behavior at any time.

## Configuration

### Enabling Review Mode

To log generated SQL to your error logs for inspection, define the following constant:

```php
define('AI_QUERY_REVIEW', true);
```

## Technical Details

- **Entry Point**: The `PlugModel::ai(string $prompt)` static method.
- **Generation**: Powered by the `QueryGenerator` service and the configured AI driver (OpenAI, Gemini, etc.).
- **Execution**: Generated SQL is passed to `QueryBuilder::rawSql()`, which safely executes the query and hydrates model instances.
