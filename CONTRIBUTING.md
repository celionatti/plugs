# Contributing to Plugs

Thank you for considering contributing to the Plugs framework!

## How to Contribute

1.  **Fork the Repository**: Create a fork of the project on GitHub.
2.  **Create a Branch**: Create a new branch for your feature or bug fix.
    ```bash
    git checkout -b feature/my-new-feature
    ```
3.  **Make Changes**: Write your code and add tests if applicable.
4.  **Commit Changes**: Commit your changes with descriptive messages.
    ```bash
    git commit -m "Add some feature"
    ```
5.  **Push to Branch**: Push your branch to your fork.
    ```bash
    git push origin feature/my-new-feature
    ```
6.  **Create a Pull Request**: Submit a pull request to the main repository.

## Coding Standards

- Follow PSR-12 coding standards.
- Write clear and concise comments.
- Ensure all tests pass before submitting.

## Reporting Bugs

If you find a bug, please open an issue on GitHub with a detailed description and steps to reproduce.

## Security Vulnerabilities

If you discover a security vulnerability, please send an email to celionatti@gmail.com instead of using the issue tracker.
